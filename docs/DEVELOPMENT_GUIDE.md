# GREATER Installer Development Guide

## Purpose

This document describes the development methodology used during the design, implementation, testing, validation, and documentation of the GREATER Installer.

The objective is to ensure that every component of the installer can be reproduced, maintained, verified, and extended by future developers and institutions.

---

## GREATER Installer Vision

The GREATER Installer is an automated deployment framework designed to reproduce the complete GREATER Ecosystem on a clean Ubuntu server with minimal manual intervention.

The installer automatically deploys and configures all major ecosystem components, including:

* Multi-Living-Lab Management
* Role-Based Access Control (RBAC)
* JWT Authentication
* Dockerized Services
* Node-RED Workspaces
* MQTT Communication
* Shared MySQL Database
* NGINX Reverse Proxy
* Monitoring and Observability Services
* Automated Configuration Generation
* Replicable Infrastructure Deployment

The objective is to enable universities, research institutions, and Living Labs to deploy the GREATER Ecosystem quickly, consistently, and reproducibly using open-source technologies.

---

## Development Workflow

Each development activity follows the same process:

1. Design
2. Implementation
3. Testing
4. Validation
5. Documentation
6. Freeze

No module is considered complete until all six stages have been successfully completed.

---

## Modules

The GREATER Installer currently consists of the following modules:

* Module 01 – Prerequisites Validation
* Module 02 – Deployment Questionnaire
* Module 03 – Configuration Generation
* Module 04 – Container Deployment
* Module 05 – Post-Installation Configuration

Additional modules may be introduced as the installer evolves.

## Documentation Standards

Every development step document must contain:

### Purpose

Explain why the activity is required.

### Why the Step is Required

Explain the problem solved by the step.

### Input Files

List all files required by the step.

### Output Files

List all files generated by the step.

### Commands Used

Document every command executed during implementation and testing.

### Expected Results

Describe the expected output after execution.

### Validation Procedure

Describe how the results were verified.

### Troubleshooting Notes

Document issues encountered and solutions applied.

### Final Status

Indicate:

* Designed
* Implemented
* Tested
* Validated
* Frozen

This documentation standard ensures that the GREATER Installer remains fully reproducible, maintainable, and understandable for future developers and institutions.
